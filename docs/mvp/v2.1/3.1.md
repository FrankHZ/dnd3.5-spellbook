## 3.1 Frontend: URL-Driven Browse State

### Goal

Browse state is:

- refresh-safe
- compatible with browser back/forward navigation
- “shareable enough” for MVP (filters encoded in URL), but not optimized for full reproducibility across different users/campaigns

### Final Decisions

**URL is the source of truth** for browse navigation filters:

- `level`
- `classIds`
- `domainIds`
- `page`

**Persisted storage is used as defaults only** when URL params are missing:

- `browseQuery` (defaults for `level/classIds/domainIds`)
- `selectedRulebookIds` (rulebook scope)

**Rulebook scope is NOT encoded in URL** (MVP decision):

- rulebook selection is treated as campaign/user configuration
- avoids huge URLs when many supplements are enabled
- avoids back-button “reverting” rulebook scope unintentionally

**Page size is fixed** (MVP decision):

- `PAGE_SIZE = 25` shared by browse + search
- not encoded in URL
- not persisted

### Implementation

#### Hook: `useBrowseQueryState`

Created a custom hook to centralize:

- parsing URL query params
- applying persisted defaults when params are missing
- canonicalizing URL (clean URLs, consistent behavior)
- updating persisted defaults on user interactions

**Rules:**

1. For each of `level/classIds/domainIds`:
   - if param exists in URL → use URL value
   - if param missing → use `state.browseQuery` default and inject into URL (replace history)

2. `page` is always URL-driven and resets to `1` whenever filters change.
3. Rulebooks come from `state.selectedRulebookIds` only.

#### Clean URL behavior

- When filters are cleared, corresponding params are removed (not set to empty strings).
- URL is updated via navigation actions rather than local component state.

#### Query integration

Browse queries use:

- effective `level/classIds/domainIds` from the hook
- `page` from the URL
- `rulebookIds` from persisted storage
- `pageSize` from shared constant `PAGE_SIZE`

### Notes / Pitfalls

- i18next extractor requires literal namespace patterns; to avoid unstable extraction, namespaces were reduced / simplified and translation calls were adjusted accordingly.
- Rulebook scope intentionally does not participate in URL history to avoid “back” reverting campaign configuration.
